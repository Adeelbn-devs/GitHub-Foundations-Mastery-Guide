# Day 16: Introduction to Code Scanning 🛡️

**Date:** May 16, 2026  
**Module:** Security and Code Scanning

## 📝 What I Learned Today:

Today, I started a brand new module focused on **DevSecOps** and **GitHub Advanced Security (GHAS)**. Finding vulnerabilities manually in a massive codebase is impossible, so automating this process is a critical skill.

### Key Objectives for this Module:
- **Understand Code Scanning:** What it is and how it proactively prevents security breaches.
- **Implementation Tools:** How to enable code scanning using **CodeQL** (GitHub's native semantic code analysis engine) and integrate third-party security tools.
- **Automation via CI/CD:** Contrasting the implementation of CodeQL within GitHub Actions versus external CI tools.
- **Configuration & Triggers:** Understanding how to configure scans based on specific triggering events (e.g., Pull Requests) versus scheduled scans (e.g., nightly cron jobs).
