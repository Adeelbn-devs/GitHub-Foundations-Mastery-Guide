# Day 2: What is GitHub & Its Core Components 🚀

**Date:** April 26, 2026  
**Module:** Introduction to GitHub

## 📝 What I Learned Today:

Today, I explored the foundational architecture of GitHub and how it differentiates from Git.

### 1. Git vs. GitHub
- **Git** is the version control system running locally to track changes.
- **GitHub** is the cloud platform built on top of Git, adding collaboration, AI, and automation.

### 2. The 5 Pillars of GitHub
- **AI:** Powered by Copilot for faster development.
- **Collaboration:** Issues, PRs, and seamless teamwork.
- **Productivity:** Automation via built-in CI/CD.
- **Security:** Integrated tools like CodeQL and Dependabot.
- **Scale:** Unmatched global developer community.

### 3. Repositories
- The central hub for a project. It stores files and their revision history.
- Learned how to **Create**, **Clone** (download locally), and **Add/Commit** new files directly via the web interface.

### 4. Gists vs. Wikis
- **Gists:** Lightweight, mini-repositories for sharing code snippets or logs. (Important: Even "Secret" Gists are accessible via URL; never store sensitive data here).
- **Wikis:** A dedicated section in a repo for long-form documentation (tutorials, architecture).

### 5. Feature Previews
- A way to test experimental GitHub features before they are released globally.
