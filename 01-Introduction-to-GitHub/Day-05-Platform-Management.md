## 📝 What I Learned Today:

Today, I explored how to manage the GitHub platform effectively to maintain productivity and avoid notification fatigue, along with an exciting feature for hosting websites.

### 1. Managing Notifications & Subscriptions
- **Automatic Subscriptions:** You are automatically subscribed to conversations if you comment, open an issue, or are `@mentioned`.
- **Finding Mentions:** Use the search qualifier `mentions:<username>` in the global search bar to find all threads needing your attention.
- **Delivery Options:** Notifications can be routed to Email, the Web Dashboard, or the GitHub Mobile App.

### 2. Filtering Notifications (Watch Settings)
You can fine-tune repository alerts using the `Watch` button:
- **Watching:** Receive alerts for all activity.
- **Not watching:** Only receive alerts when participating or `@mentioned`.
- **Custom:** Choose specific events (e.g., only Pull Requests).
- **Ignore:** Mute all notifications.

### 3. GitHub Pages
- **What it is:** A free static site-hosting service provided by GitHub.
- **How it works:** It takes HTML, CSS, and JavaScript files directly from a repository (or a specific folder like `/docs`) and publishes them as a live website. Perfect for personal portfolios and project documentation!
