# Day 11: GitHub Billing and Payments 💸

**Date:** May 11, 2026  
**Module:** Introduction to GitHub Administration

## 📝 What I Learned Today:

Today, I explored how GitHub structures its billing and licensing models. Understanding this is crucial for managing budgets in a startup or enterprise environment.

### 1. The Two Parts of a GitHub Bill
- **Subscriptions (Fixed Cost):** Recurring monthly fees for your account plan (e.g., Team, Enterprise) and fixed add-ons like GitHub Copilot.
- **Usage-based Billing (Metered):** Variable costs that scale with usage, primarily applied to GitHub Actions (runtime minutes) and GitHub Packages.

### 2. Pricing for GitHub Actions
- **Public Repositories:** 100% Free and unlimited.
- **Private Repositories:** Comes with a set number of free minutes (e.g., 2,000 mins for Free accounts). Usage beyond this requires configuring a spending limit.

### 3. Usage-based Licensing (Enterprise)
- **Monthly Billing:** Organizations only pay for the exact number of active licenses used that month.
- **Pro Rata Charges:** If a user joins mid-month, the organization is only billed for the portion of the month the license was active.

### 4. Billing Platform Features
- **Spending Controls:** Admins can set hard limits to prevent unexpected overages.
- **Alerts:** Automated email notifications trigger when approaching defined spending limits.
