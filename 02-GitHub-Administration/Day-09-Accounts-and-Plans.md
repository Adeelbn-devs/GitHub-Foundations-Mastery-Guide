# Day 9: GitHub Accounts and Plans 🚀

**Date:** May 09, 2026  
**Module:** Introduction to GitHub Administration

## 📝 What I Learned Today:

Today, I explored the core administrative structures of GitHub. It is essential to distinguish between the structural hierarchy (Accounts) and the feature/billing tiers (Plans).

### 1. GitHub Account Types (Structure)
- **Personal Accounts:** The individual user's identity. All actions (commits, PRs) are attributed to this account. You log in here.
- **Organization Accounts:** A shared workspace for multiple collaborators. You cannot log into an org; you access it via your Personal account. It features tiered permissions and team structures.
- **Enterprise Accounts:** The overarching umbrella used to centrally manage billing, policies, and multiple Organizations at a massive scale.

### 2. GitHub Plans (Billing & Features)
- **GitHub Free (Personal & Org):** Includes unlimited public and private repos, 2,000 Actions minutes (personal), and 500MB Packages storage. Private repos have limited advanced features.
- **GitHub Pro (Personal):** For solo developers needing advanced tools in private repos (Protected Branches, Pages, Wikis) + 3,000 Actions minutes.
- **GitHub Team (Org):** Designed for collaborative groups. Includes Draft PRs, multiple/required reviewers, and advanced repository insights.
- **GitHub Enterprise:** Built for scale, compliance, and security. 
  - **Cloud:** Hosted by GitHub (50,000 Actions minutes, 50GB storage, 99.9% SLA).
  - **Server (GHES):** Self-hosted for total infrastructure control.
  - *Features Enterprise Managed Users (EMU)* for central identity management.

*Note: GitHub Copilot is a separate add-on and is not included by default in any of these base plans.*
